Getting started

GP-Tool was written and tested on Linux and Windows. As I no longer posses a Mac computer, I don't have the means to tested it, but it should be possible compile and use it in principle. For these reason, in the next sections I present how to get started in Windows and Linux.

Binary packages

Installation packages generated using CPack via CMake are provided for Windows (64 bits) and Linux (Arch distribution). Both packages can be found here. The installation procedure consists of simply uncompressing the contents of this package.

I'm aware that the Windows package might run under problems concerning security, as it is an third party binary file to be executed. In some cases it is a simple matter of getting permission from the administrator to execute. Unfortunately, Windows will block independently of granted permissions in other situations. I'm studying a solution for this second issue. In the meantime, I suggested compiling GP-Tool from source as explained in the following section.

Considering the Linux binary package. It works fine under Linux distros parented into the Arch Linux family. It is possible that it might not work under the Debian family, for example. Unfortunately, for those cases, the user might need to compile GP-Tool from source code as explained in the following section.

Compiling from source code

The procedure to install GP-Tool under Linux, Windows and MacOS are very similar. In all cases, the user will need to have pre-installed a modern version CMak, an OS specific compiler and Git. CMake is an open-source cross-platform software to build programs and libraries. Popular choices of compilers are GCC/G++ for Linux, clang for MacOS and MSVC for Windows. Just be sure to have a relatively modern compiler as the standard c++17 is required for compilation. Finally, Git is a development tracker that we will use to clone the source code from GitHub.

With the list of pre-requisites fullfiled, the user navigate to any directory of preference using the the terminal and clone the GitHub repository along with all its dependencies using the following command:

git clone --recurse-submodules -j4 https://github.com/guilmont/GP-Tool.git

Once the cloning is completed, use the explorer and navigate until cloned folder. There create a folder called "build", in which CMake will do its magic. From here, the commands will differ slightly, hence I will split in subsections.

Linux and MacOS

Using the terminal, navigate until recently created bulid folder. There are only to options that are relevant for the average user, hence I'll focus on those: CMAKE_INSTALL_PREFIX and CMAKE_BUILD_TYPE. In the terminal write:

cmake .. -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=whatever_path_you_want_to_install

Then press enter. An example of path for installation would be "/home/user/Desktop/GP-Tool", so GP-Tool will be installed in the Desktop. For example...

Then, to compile and install GP-Tool:

make -j install

Like so...

Once installation is completed, simply navigate using the terminal or the explorer to the bin folder within the installation prefix and open the executable.

Hi-DPI screens

For users with HI-DPI screens, GP-Tool provides a configuration for scale up the size of all the components by a factor 2. To enable this feature, click on "Set HIDPI" under Tools in the GP-Tool menu bar.
